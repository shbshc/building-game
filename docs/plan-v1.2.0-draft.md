# 功能方块系统 Phase 2 — 实施计划（草案）

> 日期：2026-07-18  
> 引擎：Godot 4.7 (GL Compatibility)  
> 语言：GDScript  
> 版本目标：v1.2.0

---

**Goal:** 在 Phase 1 基础上扩展功能方块系统，增加逻辑运算、传感器、速度控制等高级功能。

**Architecture:** 延续 Phase 1 架构。新方块类型注册到 FuncType 枚举和 item_types.gd。行为逻辑集成到 main.gd 的 tick 循环或 block_manager.gd 的 slide_chain。

**Tech Stack:** Godot 4.7, GDScript, GL Compatibility renderer

---

## 候选功能方块

### 1. 速度控制方块
- **行为**：放置在移动方块旁边，改变其移动速度
- **变体**：加速 (0.3s)、减速 (3s)、暂停
- **数据**：speed_multiplier: float

### 2. 逻辑门方块
- **行为**：两个输入面，按逻辑输出信号
- **变体**：AND、OR、NOT、XOR
- **交互**：用移动方块"激活"输入面

### 3. 传感器方块
- **行为**：检测前方是否有特定方块
- **输出**：改变相邻移动方块方向或速度
- **变体**：检测普通/功能/空/任意

### 4. 传送方块
- **行为**：移动方块/推动链碰到时传送到配对位置
- **配对**：同色传送方块自动配对

### 5. 蓝图系统
- **行为**：框选区域 → 复制 → 粘贴
- **UI**：选择工具 + 预览

---

## 优先级建议

| 优先级 | 功能 | 理由 |
|:--:|------|------|
| 🔴 高 | 速度控制 | 简单、实用、扩展轨道玩法 |
| 🟡 中 | 传感器 | 自动化核心，但交互复杂 |
| 🟡 中 | 逻辑门 | 计算能力，但需要信号系统 |
| 🟢 低 | 传送 | 有趣但非必要 |
| 🟢 低 | 蓝图 | UI 工作量较大 |

---

## 实施方式

Phase 1 采用 **subagent-driven development**（每 Task 独立子代理 + 双审），Phase 2 建议继续使用相同流程。

每个功能方块遵循标准流程：
1. `functional_types.gd` — 注册 FuncType 枚举
2. `item_types.gd` — 注册 6 方向变体
3. `block_manager.gd` — 添加行为逻辑
4. `main.gd` — tick 集成
5. `inventory_manager.gd` — 背包初始化

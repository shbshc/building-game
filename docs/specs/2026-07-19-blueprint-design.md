# 蓝图系统 — 设计文档

> 日期：2026-07-19
> 引擎：Godot 4.7 (GL Compatibility)

## 1. 概述

手持「蓝图笔」框选建筑区域，压缩为蓝图方块。支持两种使用模式：展开还原原始建筑，或作为微型模型装饰。

## 2. 使用流程

```
手持蓝图笔 → 右键拖拽选区域 → 松开右键 → 弹出选项
  ├── 压缩为蓝图（可展开）→ 蓝图方块入背包
  └── 压缩为微型模型 → 微型方块入背包

放置蓝图方块 → 右键点击
  ├── 展开 → 原建筑块原地重建
  └── 无操作 → 作为微型模型展示 6 面投影
```

## 3. 技术方案

### 3.1 蓝图笔 (Item 36)

- 物品栏中新增 `Blueprint Tool` 条目
- 手持时右键拖拽 = 框选
- 框选高亮显示选中的长方体区域

### 3.2 选区

- 拖拽起点 → 终点确定长方体
- 实时高亮半透明蓝色立方体覆盖选中区域
- 只支持 4³ 和 8³ 两种尺寸（自动对齐到最近的合法尺寸）

### 3.3 压缩

- 扫描选区内所有方块 → 记录相对坐标、类型、功能、方向、贴图
- 生成 JSON → `user://blueprints/bp_XXXX.json`
- 创建蓝图物品 → 入背包

### 3.4 微型外观

- 创建 16×16 的 6 面投影贴图
- Godot 做法：新建一个临时 SubViewport，从 6 个方向渲染选区内方块到 16×16 RenderTarget
- 贴图贴在蓝图/微型方块的 6 个面上

### 3.5 展开

- 读取 JSON
- 按相对坐标 + 放置位置偏移，调用 place_block
- 遇到已有方块的位置跳过
- 保持原有方块的 func_type、direction、model_id

## 4. 文件结构

```
scripts/blueprint_tool.gd      ★ 蓝图笔逻辑（选区 + 压缩 + 展开）
scripts/blueprint_data.gd      ★ 蓝图数据读写
user://blueprints/             ★ 蓝图存储目录
```

## 5. 数据格式

```json
{
  "name": "小屋",
  "size": [4, 4, 4],
  "blocks": [
    {"x":0,"y":0,"z":0,"i":0,"f":0,"d":0,"m":"stone"},
    ...
  ]
}
```

## 6. 实施顺序

| 阶段 | 内容 |
|------|------|
| 1 | `blueprint_data.gd` — 数据读写 |
| 2 | `blueprint_tool.gd` — 框选 + 压缩 |
| 3 | 微型贴图生成 |
| 4 | 展开还原 |
| 5 | 物品栏接入 |
